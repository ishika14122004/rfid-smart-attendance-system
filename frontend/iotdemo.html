// Step-by-Step Flow
How Each Step Works
A detailed breakdown of every stage in the system pipeline.
1
Student Taps RFID Card
The student brings their RFID card close to the RC522 reader. The reader emits a 13.56 MHz radio frequency signal that powers the card's embedded chip and reads the unique 8-character UID stored on it.
Hardware Layer
2
ESP32 Receives & Validates the UID
The RC522 sends the UID to the ESP32 via SPI protocol. The ESP32 runs validation logic — checking if the UID is registered, if it falls within the allowed time window, and if it's a duplicate scan from the same session.
Hardware Layer
3
ESP32 Sends Data to Firebase via Wi-Fi
If the UID passes validation, the ESP32 connects to the internet over Wi-Fi and makes an HTTP POST request to the Firebase Realtime Database — sending the student UID, timestamp, and attendance status.
Cloud Layer
4
Firebase Stores the Record in Real-Time
Firebase instantly stores the attendance record and pushes the update to all connected clients. No page refresh is needed — the data appears live on the faculty dashboard the moment the card is tapped.
Cloud Layer
5
Faculty Dashboard Updates Live
The web dashboard listens to Firebase in real-time. It updates the attendance table, stat cards, and percentage bars automatically. Faculty can filter by present/absent, search students by name or reg number, and export a CSV report.
Web Layer · Currently Active
// Hardware Components
Physical Components in Final System
These are the actual hardware components that will be used once procurement is complete. The software is fully ready to interface with them.
ESP32 Dev Board
Dual-core microcontroller with built-in Wi-Fi and Bluetooth. Acts as the brain of the system — reads UID from RC522 and POSTs data to Firebase.
· Xtensa LX6 @ 240MHz · 802.11 b/g/n Wi-Fi · SPI / I2C / UART support
RC522 RFID Reader
13.56 MHz RFID reader/writer module. Detects MIFARE cards within ~5cm range and transmits the unique UID to ESP32 over SPI.
· 13.56 MHz frequency · SPI interface (up to 10Mbps) · Read range: ~5cm
MIFARE RFID Card
Each student is assigned a MIFARE Classic 1K card with a unique 4-byte UID. No battery required — powered by the reader's RF field.
· MIFARE Classic 1K · 4-byte unique UID · Passive (no battery)
About This Simulation
Since hardware procurement is pending, the current prototype simulates the ESP32 + RC522 layer through a web interface. Students enter their registration number (which maps to their RFID UID) manually — this is equivalent to the card tap event. All validation logic (duplicate detection, time-window checks) and the dashboard are fully functional and identical to what the hardware integration will produce. Once the ESP32 and RC522 are available, only the input source changes — the rest of the system remains exactly the same.
